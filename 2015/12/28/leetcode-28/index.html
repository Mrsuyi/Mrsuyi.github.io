leetcode-28 implement-strstr

发表于 2015-12-28   |   分类于 LeetCode   |  

题目地址 https://leetcode.com/problems/implement-strstr/

字符串匹配。这个问题比较经典，在此列举《算法导论》里面介绍的4种方法

朴素算法

朴素算法就是暴力搜索，逐位进行完整匹配，失败后移动一位。不需要预处理，性能很糟糕，这里就不放代码了，反正也AC不了。。。

Rabin-Karp算法

核心思想就是hash，每次取子串hash，如果hash结果匹配就逐位比较来排除伪命中。不过比普通hash好的一点是处理该问题时，前一次的hash计算结果可以被后一次hash计算有效利用，比如在“1234”中搜索“234”，假设hash函数就是str2int，前一次的结果“123”可以去除高位、加入低位计算出“234”，相当于利用了之前的信息。

KMP算法

KMP（Knuth-Morris-Pratt）算法是很经典的一个算法，网上讲解非常非常多，[这一篇]（http://www.ruanyifeng.com/blog/2013/05/Knuth%E2%80%93Morris%E2%80%93Pratt_algorithm.html）图片比较好，虽然没讲重点。。。

KMP算法和后面的有限自动机很类似，都是在向前匹配失败之后回退到一个最长前缀的位置。比如要匹配“aabaac”，而输入了“aabaa”之后最后一位失配了，只需要从“aa”继续匹配就行了。唯一区别是KMP会回退至不考虑当前输入的最长前缀然后继续比较，而自动机考虑了当前输入，更高效一些，占用空间也更多。其实没啥区别 ╮(╯_╰)╭
这里主要难点是求最长前缀数组，该数组中元素i对应子串[0,i]尾部所包含的最长前缀。比如“aabaa”：i=1时，最长前缀为“a”，数组值为1,；i=4时，最长前缀为“aa”，数组值为2。这个求解过程非常像DP，每次利用前i-1个值求解第i个值。
下面分析一个例子。

图中p代表模式，idx代表下标，prefix代表所求数组，当前位i，比较位j。prefix[j]=3说明有长度为3的最长前缀，即绿色部分是相同的，那么就比较p[i]和p[3]。如果相同，显然所求值是3+1=4。如果失败了，我们无法从末尾的绿色部分中寻找更短的前缀（因为数组只保存最长前缀），但是考虑到绿色部分是相同的，我们可以通过取j=prefix[j]-1，从首部的绿色部分中尝试寻找更短的前缀

此时prefix[j]=1说明有长度为1的最长前缀，即橙色部分是相同的，那么继续比较p[i]和p[1]，发现相同，那么所求值即1+1=2。如果不同，那么就重复j=prefix[j]-1直到j=0，这样就完成了数组的求解。
